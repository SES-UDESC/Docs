Build

Makefile#

Para faciltar o desenvolvimento e deploy, um arquivo Makefile está localizado na raiz do projeto.

Esse arquivo cria alguns aliases para comandos relacionados ao docker e ao projeto em geral.

Instalação#

  • Linux (Deb Based)
  • Windows

O make pode ser instalado em sistemas que usam apt

sudo apt-get install make

Comandos#

Todos os comandos pressupõem make como prefixo.

Exemplo:

make build
Setup#

Comandos relacionados à configuração do ambiente do projeto

ComandoDepende deDescrição
setupcreate-secretsComando utilizado para configurar o ambiente, deve ser utilizado toda vez que clonar o repositório.
create-secretsGera os arquivos secrets para o docker, onde deve ser configurado usuário e senha do banco e do servidor.
Desenvolvimento#

Comandos relacionados ao docker no ambiente de desenvolvimento

ComandoDescrição
buildConstroi as imagens, fazendo o download das imagens base, e interpretando os arquivos Dockerfile.
upSobe as imagens construidas, tornando os serviços operacionais.
downDesce e remove oa containers que o cmd up subiu, deve ser utilizado após o fim da jornada.
downvO mesmo que o cmd down, mas remove os volumes do docker, removendo possíveis informações salvas no banco.
configExibe as configurações do arquivo docker compose, resolvendo variáveis de ambiente, secrets, etc...
Produção#

Comandos relacionados ao docker no ambiente de produção

ComandoDescrição
build-prodConstroi as imagens de produção, fazendo o download das imagens base, e interpretando os arquivos Dockerfile.
up-prodSobe as imagens construidas para produção, tornando os serviços operacionais.
upd-prodSimilar ao cmd up-prod, mas não deixa o processo preso ao terminal, esse deve ser o cmd para subir os containers na produção.
down-prodDesce e remove os containers que o cmd up-prod subiu.
downv-prodSimilar ao cmd down-prod, mas remove os volumes do docker, removendo possíveis informações salvas no banco.
